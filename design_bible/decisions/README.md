# Perch Architecture Decision Records

Architecture Decision Records (ADRs) document deliberate architectural choices that affect Perch's long-term direction.

The Design Bible is the product architecture contract. ADRs explain why that contract changes or why a major interpretation was chosen.

---

## Rules

Create an ADR when a decision changes or clarifies:

- Domain ownership
- Engine responsibility
- Data model boundaries
- Source lineage rules
- Privacy or trust law
- Implementation architecture
- AI-agent workflow authority
- Chapter structure or governance

Do not create an ADR for ordinary wording cleanup, typo fixes, or implementation tasks that do not affect architecture.
