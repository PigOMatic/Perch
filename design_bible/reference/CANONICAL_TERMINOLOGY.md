# Perch Canonical Terminology

**Purpose:** Keep Perch's language stable across the Design Bible, engineering documents, code, and AI-agent work.

---

## Core product terms

| Term | Canonical meaning |
|---|---|
| Perch | A Life Operating System focused on clarity, trust, and calm daily guidance. |
| Design Bible | The authoritative product architecture contract. |
| Repository | The current implementation state. |
| Atlas | The full map of Perch domains and engines. |
| Today | The home screen and daily attention surface. |
| Read Mode | Calm default mode; shows what matters immediately. |
| Explore Mode | Deeper inspection mode; supports filtering, editing, and investigation. |
| Living World | The visual/place layer where clarity is presented. |

---

## Engine terms

| Term | Canonical meaning |
|---|---|
| Truth Engine | Determines what Perch is allowed to say. |
| Priority Engine | Determines what deserves attention. |
| Recommendation Engine | Determines what Perch should suggest. |
| Voice Engine | Determines how Perch should say something. |
| Belief Engine | Forms and ages beliefs from evidence. |
| Learning Engine | Captures personal context from user input. |

---

## Trust terms

| Term | Canonical meaning |
|---|---|
| Fact | Directly stored or observed information. |
| Derived Fact | Arithmetic or deterministic transformation over facts. |
| Interpretation | A judgment made from facts. |
| Recommendation | A suggested action. |
| Prediction | A claim about the future. |
| Source Lineage | The traceable chain showing where a claim or figure came from. |
| Confidence | How sure Perch is that a claim or feature shape is valid. |
| Status | The maturity state of a chapter, feature, or engine. |

---

## Status terms

| Status | Meaning |
|---|---|
| Concept | Believed product direction; not specified or built deeply enough yet. |
| Planning | Being specified; behavior is defined enough to guide work. |
| Prototype | Partially built or spiked; not production-complete. |
| Implemented | Shipped and working in the repository today. |

---

## Terms to avoid or use carefully

| Avoid / Use carefully | Preferred framing |
|---|---|
| Dashboard | Use only when referring to UI layout; Perch is broader than a dashboard. |
| AI memory | Prefer Knowledge, Brain, Belief, or Source Lineage depending on meaning. |
| Smart suggestion | Prefer Recommendation with evidence and reasoning. |
| Automation | Use only when an action can actually happen automatically. |
| Sync | Use only when real external data synchronization exists. |
| Search | Do not imply semantic/vector search unless implemented. |
| Integration | Do not use unless an external data path exists. |

---

## Standing language rule

When in doubt, choose the term that preserves truth over excitement.

Perch should sound calm, precise, and trustworthy.
