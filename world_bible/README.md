# Perch World Bible

## Purpose

The World Bible defines what Perch feels like to live inside.

The Design Bible answers: **How does Perch work?**  
The World Bible answers: **What world does Perch return the user to?**

Perch is not a dashboard, theme picker, or productivity app skin. Perch is a calm personal world that grows as the user's life grows.

## Core Vision

Perch is a safe place to land.

A perch is where a person can pause, look over their world, regain perspective, and decide what matters next.

The user should not feel like they opened another app. They should feel like they returned to a familiar place where their life has already been organized in front of them.

## Locked Emotional Goal

Every primary Perch experience should create this feeling:

> I know what matters.  
> I know what is safe.  
> I know what is coming.  
> I know what can wait.  
> I can breathe.

## World Model

Perch is one persistent natural sanctuary that expands over time.

The foundation is nature:

- a home perch
- a forest trail
- a creek
- a cabin
- a firepit
- a dock
- a barn/workshop
- an RV campsite
- a break room for work context

The user does not collect random themes. The user builds and expands a life.

Scenes are not cosmetics. Scenes represent chapters, contexts, priorities, seasons, and the life the user is building.

## Primary Metaphor

The scene is the interface.

Information does not float in cards. It lives on objects inside a place:

| Life tool | Object |
| --- | --- |
| Calendar | Notebook, planner, wall calendar |
| Tasks | Clipboard, checklist, index card |
| Money | Envelope, ledger, receipt stack |
| Reminders | Sticky note, pinned note |
| Weather | Window, porch, sky, creek |
| Perch Brain | Journal, field notebook, voice card |
| Work shifts | Tickets, badge cards, slips |

## Rules

1. Perch is a place, not a dashboard.
2. Nature is the foundation.
3. Life is the story.
4. The world grows with the user.
5. Core tools are never locked behind progression.
6. New scenes add meaning and atmosphere, not basic access.
7. Every object must have a reason to exist.
8. Do not make the word `Today` the visual hero.
9. Do not bake live user data into image assets.
10. User data is live UI layered onto physical-feeling objects.
11. Calm beats productivity.
12. Relief comes before explanation.

## World Expansion Principle

The user's trails expand as their life expands.

A new scene may appear because:

- the user is traveling
- the user is working nights
- the user is building a homestead
- the user pays down debt
- the user starts a family routine
- the user enters a new season of life
- the user chooses a calmer place to review a specific life area

The world should reflect reality without becoming a distracting game.

## Canonical Starting Scene

The first native prototype should start with one scene:

**Home Perch / Cabin Desk**

Objects:

- notebook for the daily run sheet
- cash envelope with money sticking out
- sticky note for the one brain item
- shift tickets tucked near the notebook
- coffee mug, pen, and natural props
- window/light cue for weather or time of day

The scene should say: your life is already laid out, and you can breathe.
