# S Flow Week Layout

Status: layout experiment
Created: July 7, 2026

The Today story demo now tests an S-shaped reading path:

1. Money moves left to right.
2. Week context moves right to left.

The oversized first-glance headline was reduced so the page opens faster and gives more space to useful information. The opening should not tell the user what is first. The layout should show importance through size, placement, and visual weight.

The money section now uses plain language:

- Money
- Open after bills
- Bills checked

Bills checked is a pressable details tab. It keeps bill rows, due dates, amounts, and check states hidden until the user asks for proof.

The week section shows the full week schedule rather than a three-day calendar strip. It is intended to orient the user without becoming a full calendar.

Color should be mostly restrained. The page can stay neutral, while the most important object receives a small accent. Color should mean attention, not decoration.

The layout can later adjust based on need, severity, or wants. A quiet Always show control represents the future ability to keep specific surfaces visible, such as Work, Bills, Kids, Projects, or Goals.

Design rules:

- Show priority through hierarchy instead of telling the user what is first.
- Use color sparingly for the thing that currently matters most.
- Let users pin recurring wants or needs without making the whole page a settings screen.
- Guide the eye through useful objects, not stacked dashboard sections.
