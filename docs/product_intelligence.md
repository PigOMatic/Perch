# Product Intelligence

Perch should improve by understanding how the product is used, not by collecting private user content.

## Core Principle

Measure behavior, not personal details.

## What Perch May Track

Anonymous or privacy-respecting usage events such as:

- app opened
- Today viewed
- Brain capture created
- recommendation expanded
- recommendation dismissed
- goal created
- bill marked paid
- search used
- screen exited
- setup step completed
- error occurred

## What Perch Should Not Collect by Default

Perch should not collect private content by default:

- Brain note text
- journal entries
- transaction descriptions
- email contents
- calendar event details
- medical information
- addresses
- photos
- passwords
- private documents

## Product Questions to Answer

- Which features are used most?
- Which features are ignored?
- Where do users quit?
- Which setup steps cause friction?
- Which recommendations are dismissed?
- Which features correlate with retention?
- Which screens create daily value?

## Feature Health

Each major feature may eventually receive a health score based on:

- discoverability
- usage frequency
- return usage
- completion rate
- satisfaction
- error rate
- retention impact

## Privacy Promise

Perch learns how the app is used, not what the user's private life contains.

## User Control

Users should eventually be able to:

- see what telemetry is collected
- opt out of product analytics
- delete analytics identifiers where practical
- understand why analytics helps improve Perch
