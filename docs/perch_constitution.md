# The Perch Constitution

## Purpose

The Perch Constitution defines the immutable truths of Perch.

It exists above product features, design explorations, visual styles, implementation details, and temporary prototypes.

If a future decision conflicts with this Constitution, the decision should be redesigned.

## The Ten Truths

### 1. Perch is a place.

Perch is not primarily a dashboard, task manager, calendar, budget app, or note app.

Perch is a calm place the user returns to when life feels scattered.

### 2. Nature is the foundation.

The identity of Perch is grounded in nature: trails, cabins, creeks, firelight, trees, rain, docks, gardens, barns, and quiet outdoor perspective.

Life contexts may enter the world, but nature remains the root.

### 3. Calm beats productivity.

Perch does not exist to help people do more for the sake of doing more.

Perch exists to help people feel less overwhelmed while doing what matters.

### 4. Information lives on objects.

Important information should feel like it belongs to the world:

- calendar in a notebook
- money in an envelope or ledger
- reminders on sticky notes
- tasks on a clipboard
- ideas in a journal
- shifts on tickets or badge cards

Floating app cards are a fallback, not the primary language.

### 5. The world grows with the user.

Users do not collect cosmetic themes.

They build and return to a sanctuary that expands as their life expands.

New places represent chapters, contexts, or meaningful priorities.

### 6. The user owns persistent possessions.

The notebook is their notebook.

The journal is their journal.

The money envelope is their envelope.

The trail map is their map.

These objects should feel familiar across days, scenes, and seasons.

### 7. Every feature must reduce mental load.

A feature that adds pressure, clutter, guilt, or decision fatigue does not belong in its current form.

Perch should clarify before it asks.

### 8. The interface should disappear into the environment.

The user should not feel like they are managing software.

They should feel like they walked up to a place where their life has already been laid out.

### 9. The user should feel more hopeful after opening Perch.

Even when life is messy, Perch should create a small path forward.

It should help the user regain footing, not remind them how far behind they are.

### 10. Perch never punishes the user for falling behind.

Perch must never shame the user.

If the user returns after a long absence, Perch should not say:

> You missed 41 tasks.

It should say something closer to:

> Welcome back. Let's get your footing again.

## Product Implications

- The word `Today` should not become the visual hero.
- The default experience should be relief, not review.
- Core utility should not be locked behind game progression.
- Progression should reflect real life, not addictive reward loops.
- Scenes should never become a noisy theme store.
- Data must stay live and trustworthy.
- The first moment should make the user breathe easier.

## Build Test

Before shipping any major feature, ask:

1. Does this make Perch feel more like a place?
2. Does this reduce the user's mental load?
3. Does this preserve calm?
4. Does this avoid shame?
5. Does this make the user's life feel more organized before they do anything?

If the answer is no, redesign it.
