# AI Contribution Rules

This file is for AI collaborators and Future Jeff.

## Before Doing Work

1. Read `perch_ai_workspace/README.md`.
2. Read `perch_ai_workspace/PROJECT_STATE.md`.
3. Read the relevant Design Bible or build-plan files.
4. Audit repository reality before making implementation claims.

## Rules

- Do not modify application code unless explicitly requested.
- Do not pretend planned features are implemented.
- Separate current reality from future vision.
- Label status honestly: Concept, Planning, Prototype, Implemented.
- Report every file changed.
- Report whether app code changed.
- Stop after completing the requested task.

## Design Bible Work

For Design Bible chapters:

- Audit first.
- Write second.
- Update the Design Bible index if needed.
- Update change logs if present.
- Be honest about implementation status.

## Build Work

For implementation work:

- Change only the requested scope.
- Avoid broad rewrites.
- Preserve trust rules.
- Add or describe verification steps.
- Report what changed and what was intentionally left unchanged.
