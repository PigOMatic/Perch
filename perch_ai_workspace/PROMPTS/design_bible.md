# Design Bible Prompt Pattern

Read:

- `perch_ai_workspace/README.md`
- `perch_ai_workspace/PROJECT_STATE.md`
- `perch_ai_workspace/GLOBAL_LAWS.md`
- `perch_ai_workspace/STATUS_DEFINITIONS.md`

Then audit the repository for the requested chapter.

Do not modify app code.

Write the chapter only after auditing.

Every chapter must distinguish:

- Vision
- Planned architecture
- Current implementation
- Status
- Confidence
- Dependencies
- Current data sources
- Missing dependencies
- Future build order

When finished, report files changed and whether app code was modified.
