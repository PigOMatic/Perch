# Perch Art Bible

## Purpose

The Art Bible defines the visual and sensory language of Perch.

Perch should feel like a calm natural sanctuary with useful life objects placed in it. It should not feel like a corporate dashboard, generic wellness app, or game UI.

## Visual North Star

A user opens Perch and feels like they are sitting somewhere real.

They see a familiar place. Their notebook is open. Their money envelope is nearby. One sticky note is pinned where it belongs. The next shift or appointment is on a physical slip. The scene is calm before it is informative.

## Camera Rules

Preferred camera:

- 35–45 degree downward angle
- slight perspective
- table or surface visible
- foreground objects overlap
- natural imperfections
- no flat webpage composition

Avoid:

- straight-on app panels
- centered hero text
- equal-width stacked cards
- floating glass UI as primary structure
- excessive symmetry

## Lighting Rules

Lighting should communicate time, safety, and context.

Examples:

- Cabin desk: morning sunlight, warm highlights
- Creek: soft daylight, green reflections
- Firepit: warm orange edge light, dark surroundings
- Rainy window: cool diffuse light, protected interior
- Break room: muted fluorescent, night-shift calm
- RV campsite: lantern/fire light and early evening warmth

## Material Language

Preferred materials:

- wood
- paper
- canvas
- leather
- metal clips
- glass window
- water
- cloth
- notebook binding
- worn edges

Avoid overusing:

- glassmorphism
- neon gradients
- sterile white cards
- glossy SaaS surfaces

## Object Rules

Every object must have a job.

A coffee mug can create warmth. A pen can point the eye. A notebook can hold the daily run sheet. An envelope can hold money safety. A sticky note can hold one unresolved brain item.

Decorative objects are allowed only if they improve place, scale, or emotional grounding.

## Typography

Perch can use two broad text voices:

1. **Calm system text** for labels and data.
2. **Human handwritten text** for notes, notebook markings, and personal reminders.

Never make typography feel like a corporate presentation.

The word `Today` should not become the hero. The scene and user's life should be the hero.

## Motion Rules

Motion should be subtle and ambient.

Good motion:

- coffee steam
- fire flicker
- creek movement
- light through leaves
- rain on window
- paper settling
- envelope opening
- sticky note peel/flip
- notebook page turn

Bad motion:

- bouncing badges
- gamified explosions
- constant loops that compete with attention
- confetti as a default reward

## Audio Rules

Audio should be optional, gentle, and scene-specific.

Examples:

- creek water
- fire crackle
- rain
- wind
- birds
- hospital HVAC
- RV campsite ambience

Audio must never be necessary to understand information.

## Image Asset Rules

Scene assets may be AI-generated, photographed, illustrated, or blended, but live user data must remain layered UI.

Do not bake personal information into images.

Asset layers should support:

- background environment
- table/surface
- primary object layer
- foreground props
- light overlays
- shadow overlays
- live text overlays
- animation hooks

## Emotional Test

Before accepting any scene or object, ask:

1. Does this make the user feel safer?
2. Does this make the user's life feel clearer?
3. Does this feel like Perch, not a generic productivity app?
4. Would this still feel calm after opening it every day?
5. Does the user understand what matters without reading a manual?
